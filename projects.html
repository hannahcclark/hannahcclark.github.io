SiMBTA

A simulation of the MBTA's Red Line in Erlang, November, 2014

This was completed as a final project for Concurrent Programming in collaboration with Raewyn Duvall and Andrew Stephens. This project allows for the simulation of the Ashmont direction of the Red Line, where users may input varying numbers of passengers, schedule trains, and force delays on trains then view the result to analyze the effects on travel times. While it likely does not follow best practices for Erlang, it was an interesting exercise in concurrent simulation and is of reasonable quality given the time period available for the project.

CollaboRead

An iPad application to improve student engagement in radiology lectures, September, 2014 - Current

This project serves as my capstone project for my degree through the Tufts School of Engineering. It was initially designed by Emily Carlin, Riley Jack Meehan, Aaron Tietz, and Shiwan Zuo, as part of a class with Ben Shapiro, the advisor for this project. The programming was done in collaboration with Andrew Dempsey and Hamid Mansoor. We have been working with the radiology department at Tufts Medical Center to best meet their needs to improve the quality of and student engagement in radiology lectures given to medical students on rotation. The app allows students to analyze images provided by lecturers and identify abnormalities and lecturers to display the responses. The goal of the app is to increase the amount of students participating in each lecture and to spark discussions between students as well as between the students and lecturers.

Color-Assignment-Generator

A Node.js module that generates as many vibrant, unique, and visibly different RGB colors as possible, April, 2015 - Current

This was initially developed as a utility for Collaboread. We needed to be able to assign users unique colors to use to mark up images. However, many of the color generators available included colors that were not easily visible on radiological images. We also needed to be able to quickly and easily recover which colors had already been used incase the server went down. Thus Color-Assignment-Generator was born. It generates colors using HSV to be able to guarantee the appropriate vibrance for visibility then converts them to RGB. Colors become closer to already generated ones and later may repeat as more are generated, due to the limited number of visually distinct colors. Settings can be changed to generated more or fewer new colors before repeating. In addition, the settings and current stage of generation may be retrieved as an object that can be saved or used to create a new color generator that will start generated where the original one left off.
